# Telegram Vote Bot

Бот для создания кастомных опросов в Telegram:

- обычные и анонимные опросы;
- один или несколько вариантов ответа;
- постер-картинка к опросу;
- кнопки голосования под сообщением;
- результаты в процентах;
- список проголосовавших по каждому пункту для неанонимных опросов.

## Запуск

1. Создайте бота через [@BotFather](https://t.me/BotFather) и получите токен.
2. Для публикации в каналы и удобной отправки в другие чаты включите inline mode у бота через BotFather: `/setinline`.
3. Установите зависимости:

```bash
yarn install
```

4. Создайте `.env`:

```bash
cp .env.example .env
```

5. Укажите `BOT_TOKEN` в `.env`.
6. Запустите бота:

```bash
yarn dev
```

## Команды

- `/start` или `/help` - подсказка.
- `/newpoll` - мастер создания опроса.
- `/cancel` - отменить создание.

Создавать опрос можно прямо в том чате, где он должен появиться. Для групп добавьте бота в группу и выдайте ему право отправлять сообщения. Если у бота включён privacy mode, отвечайте именно на сообщения-мастера бота; он включает режим ответа автоматически.

## Сценарий создания

1. Отправьте `/newpoll`.
2. Введите вопрос.
3. Введите варианты ответов, каждый с новой строки.
4. Настройте анонимность, мультивыбор и постер.
5. Опубликуйте опрос.

Если включена анонимность, бот показывает только количество и проценты. Если анонимность выключена, в результатах будут имена проголосовавших по каждому варианту.

## Публикация в другом чате или канале

Самый удобный способ - кнопка `Поделиться` под опросом. Она открывает выбор чата или канала и отправляет туда рабочую inline-копию опроса.

Для канала:

1. Включите inline mode у бота через BotFather.
2. Нажмите `Поделиться` под опросом.
3. Выберите канал.
4. Опубликуйте предложенную карточку опроса.

Люди смогут голосовать прямо в канале через кнопки. Все inline-публикации используют одни и те же голоса и обновляются вместе.

Команда `/sharepoll <код>` остаётся запасным способом для групп и чатов, где бот уже добавлен.

## Хостинг

Боту нужен постоянно работающий Node.js worker. Если выключить процесс, кнопки в Telegram останутся, но голоса и результаты не будут обрабатываться.

### Railway

Самый быстрый вариант для старта:

1. Создайте проект из GitHub-репозитория.
2. Укажите переменные окружения:
   - `BOT_TOKEN`
   - `DATA_FILE=/app/data/db.json`
3. Start command:

```bash
yarn start
```

4. Build command:

```bash
yarn install --immutable && yarn build
```

Для сохранения голосов после redeploy подключите volume к `/app/data`.

### Render

В проект добавлен `render.yaml` для worker-сервиса. В Render нужно добавить secret `BOT_TOKEN`.

Если используете JSON-хранилище, подключите persistent disk или перенесите хранение в SQLite/Postgres.

### VPS

Для домашнего сервера, выделенного компьютера или VPS используйте systemd. Подробная инструкция лежит в [deploy/SERVER.md](deploy/SERVER.md).

Если проект лежит в GitHub/GitLab, обновление на сервере можно делать одной командой:

```bash
./deploy/update.sh
```

Скрипт делает бэкап `data/db.json`, подтягивает `git pull --ff-only`, ставит зависимости, собирает `dist` и перезапускает `pm2`.

Чтобы запускать обновление из любой папки короткой командой:

```bash
sudo ./deploy/install-update-command.sh
vote-bot-update
```

Короткая схема:

```bash
yarn install --immutable
yarn build
BOT_TOKEN=... DATA_FILE=./data/db.json yarn start
```

Для текущей версии бота открытый 80 порт не нужен: бот работает через long polling и сам подключается к Telegram. Серверу нужен исходящий HTTPS-доступ к `api.telegram.org`; для администрирования обычно нужен SSH.

На VPS данные в `data/db.json` будут жить на диске сервера, но всё равно стоит делать бэкап.

### Docker

В проект добавлен `Dockerfile`:

```bash
docker build -t vote-bot .
docker run -d --name vote-bot \
  -e BOT_TOKEN=... \
  -v vote-bot-data:/app/data \
  vote-bot
```
